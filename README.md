# Push Notification Logger

Приложение для Android на Kotlin, которое перехватывает все входящие пуш-уведомления, сохраняет их в локальную базу данных SQLite и предоставляет удобный интерфейс для просмотра истории уведомлений.

## Возможности

- ✅ Перехват всех входящих уведомлений системы
- ✅ Сохранение уведомлений в локальную базу данных (Room)
- ✅ Просмотр истории уведомлений с сортировкой по времени
- ✅ Отображение заголовка, текста, пакета приложения и времени получения
- ✅ Очистка всей истории уведомлений
- ✅ Современный интерфейс на Jetpack Compose
- ✅ Material Design 3
- ✅ Абстрактная векторная иконка

## Требования

- Android 8.0 (API 26) и выше
- Android Studio Hedgehog или новее
- Gradle 8.13
- Kotlin 2.1.0

## Установка и запуск

### 1. Клонирование проекта

```bash
git clone <repository-url>
cd push-logger
```

### 2. Сборка приложения

#### Отладочная версия (Debug)

```bash
./gradlew assembleDebug
```

APK файл будет создан в `app/build/outputs/apk/debug/app-debug.apk`

#### Релизная версия (Release) с подписью

Для сборки подписанной релизной версии необходимо настроить переменные окружения:

1. **Создайте Keystore** (если еще не создан):
   ```bash
   keytool -genkey -v -keystore release.keystore -alias your_alias -keyalg RSA -keysize 2048 -validity 10000
   ```

2. **Настройте GitHub Secrets** в настройках репозитория:
   - `KEYSTORE_BASE64` - Base64 закодированный файл keystore (для CI/CD)
     ```bash
     base64 release.keystore | tr -d '\n'
     ```
   - `KEYSTORE_PASSWORD` - пароль от keystore
   - `KEY_ALIAS` - алиас ключа
   - `KEY_PASSWORD` - пароль от ключа

3. **Соберите релизную версию локально**:
   ```bash
   export KEYSTORE_PATH=./release.keystore
   export KEYSTORE_PASSWORD=your_keystore_password
   export KEY_ALIAS=your_alias
   export KEY_PASSWORD=your_key_password
   
   ./gradlew assembleRelease
   ```

Подписанный APK файл будет создан в `app/build/outputs/apk/release/app-release.apk`

### 3. Установка на устройство

```bash
adb install app/build/outputs/apk/debug/app-debug.apk
```

Или просто откройте APK файл на устройстве.

### 4. Предоставление разрешений

После первого запуска необходимо предоставить разрешение на доступ к уведомлениям:

1. Откройте приложение
2. Нажмите кнопку "Предоставить разрешение"
3. В системных настройках найдите "Push Notification Logger"
4. Включите переключатель "Разрешить доступ к уведомлениям"
5. Вернитесь в приложение

## Использование

1. После предоставления разрешений приложение автоматически начнет перехватывать все входящие уведомления
2. Главная страница отображает список всех полученных уведомлений в обратном хронологическом порядке
3. Каждое уведомление показывает:
   - Иконку приложения-отправителя (если доступна)
   - Заголовок уведомления
   - Текст уведомления
   - Название пакета приложения
   - Время получения
4. Кнопка "Очистить историю" позволяет удалить все сохраненные уведомления

## Технические детали

### NotificationListenerService

Приложение использует `NotificationListenerService` для перехвата уведомлений. Этот сервис получает колбэки при публикации новых уведомлений системой.

### Room Database

Для хранения уведомлений используется библиотека Room от Android Jetpack:
- `NotificationEntity` - модель данных уведомления
- `NotificationDao` - интерфейс для CRUD операций
- `NotificationDatabase` - конфигурация базы данных

### Jetpack Compose

Весь интерфейс реализован с использованием декларативного UI фреймворка Jetpack Compose:
- Material Design 3 компоненты
- Темная/светлая тема
- Анимации и переходы
- Ленивая загрузка списков (LazyColumn)

## Известные ограничения

- Приложение не может перехватывать уведомления, полученные до его установки
- Некоторые системные уведомления могут быть недоступны из соображений безопасности
- На некоторых устройствах производители могут ограничивать работу `NotificationListenerService`
- Приложение должно работать в фоне для непрерывного перехвата уведомлений
